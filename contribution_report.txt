CONTRIBUTION REPORT
===================
Project : Continium
Date    : 2026-03-05
Author  : (session contributor)


OVERVIEW
--------
Full audit and repair of the frontend-backend integration for the Continium
goal-tracking app. Work covered API contract alignment, backend security,
runtime bug fixes, Swagger documentation, and server configuration.


1. FRONTEND API FIXES  (frontend/js/)
--------------------------------------
- auth.js
    * Fixed register() request body: renamed "name" to "full_name", removed
      non-existent "re_password" field.
    * Fixed auto-login after register: API returns UserOut (not tokens), so
      login() is now called explicitly to obtain tokens.
    * Fixed logout(): removed call to non-existent POST /auth/logout endpoint;
      now clears localStorage directly (synchronous).

- auth-pages.js
    * Updated handleRegister() call to match new register() signature
      (confirmPassword no longer sent to API).

- goals-list.js
    * Fixed all goals endpoints to use correct trailing-slash paths (/goals/).
    * Fixed fetchGoal(): was calling non-existent GET /goals/{id} workaround;
      restored to direct endpoint after it was added to the spec.
    * Fixed createGoalCard() field names: replaced non-existent fields
      (progress, timeSpent, daily_target_hours, color) with actual GoalOut
      fields (is_complete, duration_min).

- statistics.js
    * Removed invalid ?period= query param from GET /stats/overall.
    * Fixed fetchBreakdown(): replaced non-existent /stats/breakdown with
      correct /stats/overall-by-type?type= endpoint.
    * Added transformOverallOut() helper to convert API response shape
      {total_stats: {name: minutes}} into display-ready format; values
      multiplied by 60 since display functions expect seconds.

- layout.js
    * Fixed handleAddGoalSubmit(): remapped form fields to GoalCreate schema
      (type, duration_min, start_date, deadline, frequency) and dropped
      fields not in the schema (color, frequency_type, session_count, etc.).
    * Fixed handleProfileSave(): removed call to non-existent PUT /auth/me;
      now updates user data in localStorage only.

- route-protection.js
    * Fixed handleLogout(): removed async .then()/.catch() wrapper since
      authService.logout() is now synchronous.


2. BACKEND SECURITY FIXES  (backend/app/)
------------------------------------------
- services/goal_service.py
    * get_goal(): added ownership check — was silently returning any user's
      goal by ID (data leak). Now raises 404 if goal.user_id != user.id.
    * get_by_name(): passed user_id to DAL — was returning goals from all
      users in search results (data leak).

- db/dal/goal.py
    * get_by_name(): added user_id parameter and WHERE Goal.user_id == user_id
      filter to scope search to the requesting user only.

- db/dal/stats.py
    * aggregate_overall_for_user(): removed broken range_type parameter, fixed
      query to SUM(duration_minutes) filtered by Stats.user_id (was counting
      rows via a broken join on a non-existent "timestamp" column).
    * Added get_stat_by_goal_and_date(): function was called in add_stat() but
      never existed — caused AttributeError on every POST /stats/goal/{id}.

- services/stat_service.py
    * add_stat(): fixed wrong field name stat_data.duration ->
      stat_data.duration_minutes (matching StatCreate schema); fixed update
      path to pass explicit dict instead of mutating the Pydantic model.
    * Added missing Dict import (used in get_overall_stats_by_type but not
      imported — caused NameError on GET /stats/overall-by-type).

- services/auth_service.py
    * register(): fixed type annotation UserSignupRequest -> UserCreate
      (matching what the route actually passes).
    * register(): replaced non-existent DAL calls user.get_user_by_email()
      and user.create_user() with correct user.get_by_email() and
      user.create().
    * Removed unused UserSignupRequest import; added UserCreate import.


3. SWAGGER / API DOCUMENTATION  (backend/app/api/routes/)
----------------------------------------------------------
- Added full Markdown docstrings to every route handler across:
    * auth.py   (login, register, get_me)
    * goals.py  (create, list, get, update, delete, complete, by-date,
                 filter-by-completion, search)
    * stats.py  (add_stat, get_goal_stats, overall, overall-by-type,
                 by-date-range)
- Each docstring includes: description, auth requirement, parameter tables,
  request body tables, example response shapes, and HTTP status codes.
- Fixed route registration order in goals.py and stats.py: static paths
  (/search, /overall, /filter-by-completion) moved before dynamic path
  parameters (/{goal_id}) to prevent FastAPI swallowing them.


4. SERVER / CONFIGURATION FIXES
---------------------------------
- backend/main.py
    * Added CORSMiddleware (was missing entirely — caused "Failed to fetch"
      on every browser request due to CORS policy).
    * Added lifespan() startup handler that runs Base.metadata.create_all,
      so the SQLite database and all tables are created automatically on
      first run.
    * Added import of app.db.models so all ORM models are registered with
      Base before create_all is called.

- backend/app/main.py
    * Deleted — was a duplicate entry point with incorrect import paths,
      causing confusion about which file to run.

- backend/app/core/config.py
    * Expanded default CORS origins to include localhost and 127.0.0.1 on
      ports 3000, 5000, and 5173 (all common dev server ports).

- requirements.txt / venv
    * Replaced wrong package jwt==1.4.0 with PyJWT==2.11.0. The codebase
      uses PyJWT's API (jwt.encode, jwt.decode, jwt.ExpiredSignatureError)
      which the "jwt" package does not provide, causing AttributeError on
      every token operation.

2. DOCKER CONTAINERISATION
---------------------------
- backend/Dockerfile
    * Python 3.12-slim base image.
    * Installs from requirements.txt, copies backend/ as working dir.
    * Runs uvicorn on port 8000. /data directory created for SQLite volume.

- docker-compose.yml
    * Local dev setup: builds the backend image, mounts a named volume
      for the SQLite database, loads .env for config.

- .dockerignore
    * Excludes frontend/, mockserver/, venv/, .env, __pycache__ to
      keep the image lean.


3. CI/CD — GITHUB ACTIONS
--------------------------
- .github/workflows/ci.yml
    * Triggers on every PR to dev or main.
    * Installs dependencies + pytest, runs the test suite.

- .github/workflows/deploy.yml
    * Triggers on every push to main.
    * Builds the Docker image and pushes it to GitHub Container Registry
      (ghcr.io/wlvumar/continium-backend).
    * SSHes into the DigitalOcean droplet, pulls the new image, and
      restarts the container.

Required GitHub secrets set: SSH_PRIVATE_KEY, SSH_USER, SERVER_IP,
GHCR_TOKEN.
